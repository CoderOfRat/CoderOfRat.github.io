ES6 中的类和对象

面向对象的思维特点：

  1. 抽取（抽象）对象共用的属性和行为组织（封装）成一个类（模板）

  2. 对类进行实例化，获取类的对象

对象

在JavaScript中，对象是一组无序的相关属性和方法的集合，

类

在ES6中新增了类的概念，可以使用class关键字声明一个类，之后可以用类来实例化对象。

类抽取了对象的公共部分，他繁殖某一大类

对象特指某一个，通过类实例化一个具体的对象

面向对象的思维特点：

  1. 抽取（抽象）对象共用的属性和行为组织（封装）成一个类（模板）
  2. 对类进行实例化，获取类的对象

创建(封装)类

语法

// 创建类
class Star {
    // 构造函数 参数的传递 
    constructor(uname) {
        this.uname = uname;
    }
}
// 利用类创建对象
var ldh = new Star("刘德华");// 实例化过程
console.log(ldh.uname);

constructor 构造函数 会在实例化的过程，自动调用

即 new Star(“刘德华”);

new 创建对象的时候，constructor函数在调用结束以后，会默认返回一个对象，如果在constructor内手动return一个简单数据类型，则还是返回创建的对象，如果return一个对象，则返回此对象。

当constructor被调用的时候，首先会在内部创建了一个空对象，再把空对象指向this

class Person {
    constructor(nam,age){
        this.name = nam;
        this.age = age;
    }
}
console.log(new Person("Coder Rat",18));
// 上述过程等同于 下面的执行过程
class Person {
    constructor(){
        var o = {};
        this = o;
        this.name = "Coder Rat";
        this.age = 18;
        return this;
    }
}
class Person {
    constructor(nam, age) {
        this.name = "Coder Rat";
        this.age = 18;
        this.uname = "我是构造函数中的uname,我才是大名，我叫Nick，别人说我叫什么，我不听！"; 
        this.run = function() {
            console.log("I can run");
        }
        this.sing = function() {
            console.log("我是构造函数中的sing,我最能唱，所以其他人不准唱！");
        }
    };
    sing() { // 如果在构造器中也有同名函数，则此函数被覆盖
        console.log("I am singing a song");
    };
    uname = "jack"; // 会直接
}
var o = new Person();
console.log(o);
o.sing();
o.run();

继承

        this.hoby = "eat";
    }
    job = function () {
        alert("我是科学家");
    };
}
new Father().job();
class Son extends Father {

}
console.log(new Son().name,new Son().age,new Son().hoby,new Son().job);
/* 继承过程中，如果子类没有手动书写constructor函数那么js会默认帮我们创建constructor函数，并且会在其中
        调用super方法 */

super 既是函数 也是对象

作为函数使用的时候。可以帮助子类继承父类的方法和属性

作为对象使用的时候，则代表当前子类的父类，可以直接调用父类的方法

constructor 里面的this【指向的是 创建的实例对象】

构造函数和原型

利用构造函数创建对象

function () {

}

构造函数是一种特殊的函数，主要用来初始化对象，即为对象成员变量

构造函数与普通函数的区别

new 本质上是调用函数的方法之一

一个函数如果被new调用，那么这个函数就是构造函数

不能通过构造函数来访问实例成员，只能通过实例化的对象来访问。

静态成员只能通过构造函数访问，不能通过实例对象访问

构造函数原型 prototype

函数都拥有 prototype

prototype上定义的大部分都是函数，这样可以让每一个实例对象都共享这些函数

实例对象在访问方法的时候，首先在自己身上找，如果没有，就去到他的构造函数的prototype上查找，如果构造函数中的prototype上有 就访问此方法 否则继续向上查找

constructor本质上是保存在构造函数的原型对象上的

实例对象可以通过访问他的__ proto __属性上的constructor属性来找到他的构造函数

function Fn() {
    this.uname = "Coder Rat";
}
var fn =  new Fn();
console.dir(Fn);
console.log(fn.__proto__.constructor === Fn.prototype.constructor);

console.dir(fn);
