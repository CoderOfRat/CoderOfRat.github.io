微信小程序开发

小程序特点：体积小、方便获取与传播

2017-1月发布

小程序有自己的独立入口，可定制化更强，可以模拟APP功能

适合做用完即走的应用：

如：外卖、代驾、小游戏、报名入口

不适合做重度依赖的应用

如：大型游戏、视频播放、音乐播放器

小程序组件

组件式视图层的基本组成单元，它自带一些功能与微信风格一致的样式。

一个组件通常包括开始标签和结束标签，属性用来修饰这个组件，内容在两个标签之内。

组件名称都为小写

小程序API

1.事件监听类 API

这类API 以 on 开头，用来监听某个事件时候触发

举例：wx.onNetworkStatusChange(function callback)

2.同步API

Sync结尾

3.异步API

wx 类似于 window对象

小程序项目的结构

对于小程序来说

app.js 和 app.json 文件是必须的

对于小程序页面来说

.wxss 和 .js 文件是必须的

设置小程序项目的默认首页

app.json全局配置文件中，pages数组的第一项路径，即为项目的默认首页

UI组件化的好处

便于维护，易于扩展，减少DOM操作

声明变量： var let const 禁止使用var 能用const的不用let

页面更新前 （dom-diffrent - 判断，对比前后，发生变化的跟随变化渲染，只更新发生变化的） 页面更新后 (局部更新，减少DOM操作)

小程序常见的UI组件

text 文本组件

image图片组件

.wxss

自定义属性 data-开头

对比原生的属性选择器

设计稿 基准 375px iphone6

设计师的视觉稿一般为2倍图，也就是 iphone6作为视觉稿的标准，就可以直接以设计稿的宽高值加上rpx进行设计

图片格式一般为png webp
